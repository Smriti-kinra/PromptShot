# PromptShot Feature Catalog & Specifications

This document catalogs and details the key product features of **PromptShot**, detailing their interactive interfaces, backing state machinery, and backend server endpoints.

---

## 1. Core Game Flow & Daily Puzzles

PromptShot operates as a single-page state machine centering around a daily rotating prompt challenge.

* **Main Coordinator**: Managed via a single `gameState` hook in [App.tsx](file:///Users/smriti/Documents/GitHub/promptshot/src/App.tsx).
* **Daily Puzzle Rotation**: Sourced using the `getTodaysChallenge` utility in [challenges.ts](file:///Users/smriti/Documents/GitHub/promptshot/src/data/challenges.ts#L280). Puzzles rotate automatically based on the day of the year:
  $$\text{Index} = \text{Day of Year} \pmod{\text{Total Challenges}}$$
* **Interactive Challenges Screen**: Implemented in [ChallengeScreen.tsx](file:///Users/smriti/Documents/GitHub/promptshot/src/screens/ChallengeScreen.tsx). Displays the specific challenge guidelines, target text requirements, and an input textarea for drafting prompts.

## 2. Real-Time Scorer & Guardrail Pipeline

Prompt attempts are processed through a multi-stage validation and scoring pipeline hosted as a Supabase Edge Function in [index.ts](file:///Users/smriti/Documents/GitHub/promptshot/supabase/functions/server/index.ts).

### A. Programmatic Input Guardrails
To optimize server resources, inputs are checked before invoking the LLM sandbox:
* **Short/Generic Filter**: Rejects prompts under 10 characters or containing generic chat words (`"hi"`, `"test"`).
* **Plagiarism Guardrail**: Implemented via [isCopyPasteAttempt](file:///Users/smriti/Documents/GitHub/promptshot/supabase/functions/server/index.ts#L369). Calculates a Jaccard overlap ratio on words $>4$ characters between the user prompt and target output. Overlaps $>60\%$ trigger an immediate `0/100` score with an anti-cheating warning.

### B. Split-Execution AI Evaluation
To prevent meta-instruction prompt jailbreaks, execution is isolated from grading:
* **Isolated Sandbox**: Executes the prompt inside strict `<player_prompt>` boundaries at `temperature: 0.0` and `seed: 42` to produce deterministic raw output text ([runSandbox](file:///Users/smriti/Documents/GitHub/promptshot/supabase/functions/server/index.ts#L120)).
* **Strict Evaluation Judge**: Grades the raw sandbox output text against the static target text using structured JSON properties ([runJudge](file:///Users/smriti/Documents/GitHub/promptshot/supabase/functions/server/index.ts#L173)).
* **Brevity & Token Economy Scoring**: Programmatically compares the player's prompt size to the ideal reference baseline (`idealPrompt`), allocating up to 30 points.
* **Score Scaling**: Scales Format and Brevity scores by `Math.max(0.4, Accuracy / 50)` to penalize irrelevant or nonsense inputs.

## 3. Results Screen & Visual Metrics

Once scored, the client displays a comprehensive performance card via [ResultsScreen.tsx](file:///Users/smriti/Documents/GitHub/promptshot/src/screens/ResultsScreen.tsx).

```
   ┌────────────────────────────────────────────────────────┐
   │                    Results Screen                      │
   ├────────────────────────────────────────────────────────┤
   │  [Score Breakdown Indicator] ──► Amber (#F59E0B)       │
   │  - Accuracy: 42/50, Format: 16/20, Brevity: 22/30      │
   │                                                        │
   │  [Word-Level Diff View] ──► Color-coded text matches   │
   │                                                        │
   │  [Environmental Impact Explainer] ──► Teal (#0EA79A)   │
   │  - Water Evaporated: 11ml                              │
   │  - Carbon Emitted: 0.12g                               │
   └────────────────────────────────────────────────────────┘
```

* **Interactive Score Indicator**: An animated circular progress wheel displaying accuracy, structural formatting, brevity, and adjusted difficulty totals.
* **Word-Level Diff Analysis**: Renders differences via [PromptDiff.tsx](file:///Users/smriti/Documents/GitHub/promptshot/src/components/PromptDiff.tsx). Uses a Longest Common Subsequence (LCS) algorithm to color-code exact matches, edits, insertions, and omissions, highlighting where the sandbox output drifted from the target.
* **Live Sandbox Output Viewer**: Renders the exact, raw text generated by the model during the sandbox call.

## 4. Ecological Impact Visualizers

PromptShot tracks and renders the resource footprint of LLM queries using HSL-mint/teal styling (`var(--ps-teal)`).

* **Water Glass Visualizer**: Implemented in [WaterGlass.tsx](file:///Users/smriti/Documents/GitHub/promptshot/src/components/WaterGlass.tsx). Renders an animated SVG water glass that dynamically fills or empties, illustrating the exact milliliters of cooling water evaporated by the query.
* **Ecological Stats Cards**: Evaluated via [estimateResources](file:///Users/smriti/Documents/GitHub/promptshot/supabase/functions/server/index.ts#L67) and rendered in [ImpactRow.tsx](file:///Users/smriti/Documents/GitHub/promptshot/src/components/impact/ImpactRow.tsx). Shows comparison percentages against the daily benchmark footprint.
* **Impact Detail Modals**: Created via [ImpactExplainerModal.tsx](file:///Users/smriti/Documents/GitHub/promptshot/src/components/impact/ImpactExplainerModal.tsx) to provide scientific citations explaining how carbon and water footprints correlate directly with token calculations.

## 5. Leaderboard & Auth Gate

PromptShot features ranking tables integrated with a Supabase database.

* **Leaderboard Panels**: Implemented in [LeaderboardScreen.tsx](file:///Users/smriti/Documents/GitHub/promptshot/src/components/leaderboard/LeaderboardScreen.tsx) and [LeaderboardModal.tsx](file:///Users/smriti/Documents/GitHub/promptshot/src/components/LeaderboardModal.tsx). Displays community standings, top-scoring attempts, daily streaks, and completion counts.
* **Streaks Tracking**: Uses [streak.ts](file:///Users/smriti/Documents/GitHub/promptshot/src/lib/streak.ts) to manage consecutive daily completions, displaying a streak score with the fire emoji (`🔥`) in the top bar.
* **Auth Gate Screens**: Built inside [GateScreen.tsx](file:///Users/smriti/Documents/GitHub/promptshot/src/components/leaderboard/GateScreen.tsx). Standardizes user flows for onboarding, profile creation, credentials validation, and guest session creation.

## 7. Interactive Audio & Aesthetic Polishing

A dynamic sound engine and animation system enhance the game-like experience:

* **Procedural Sound Synthesizer**: Configured in [sounds.ts](file:///Users/smriti/Documents/GitHub/promptshot/src/lib/sounds.ts). Uses the Web Audio API to synthesize interactive interface sounds (e.g. click ticks, scan hums, and success chimes) on the fly, avoiding heavy media asset payloads.
* **Aesthetic Particle Celebrations**: Instantiates interactive 2D canvas particle explosions when a player obtains a score $\ge 90/100$.
* **Vector-Driven Visuals**: Implemented via glassmorphic design systems in [theme.css](file:///Users/smriti/Documents/GitHub/promptshot/src/styles/theme.css) featuring **Space Grotesk** for typography and **JetBrains Mono** for code inputs.
* **Safe Sandbox Storage**: Utilizes [safeStorage.ts](file:///Users/smriti/Documents/GitHub/promptshot/src/lib/safeStorage.ts) to fall back gracefully to transient in-memory buffers when cookie permissions are restricted inside web views.
